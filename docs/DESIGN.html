AI/Vector

Solution Design
v0.9 · C4 model

— How AI Vector works, from system context down to data contracts.

A daily AI newsletter with a financial-services lens — curated, not aggregated. Each morning the pipeline reads ~60 sources, deduplicates cross-posts, scores stories against an editorial rubric, drafts the ones that earn a slot, and renders a single readable HTML issue. Arman reviews every draft and ships when he's happy. Nothing auto-publishes, by design.

Governing principle
No Token Wasted

The LLM does the judgment work — what matters, what to say, what to frame as the Pulse. Code does everything else — fetching, parsing, grouping, rendering, scheduling. We never spend LLM tokens or accept LLM non-determinism on work that plain code can do reliably.

3. The Pipeline C4 Level 3

Five stages, run in order. Each reads its predecessor's file and writes its own — that's the handoff contract that lets you re-run any subset cheaply. The split between deterministic stages and LLM stages is the “No Token Wasted” principle made structural.

Figure 3 · Pipeline stages (red = LLM judgment, dark = deterministic code)

fetch

Reads config/sources.yaml; pulls ~60 feeds in parallel via feedparser + httpx; normalises each entry into an Item; exact-URL deduplicates. Writes items.jsonl + source_health.json. No LLM. ~15–30s.

cluster

Embeds each item locally with BAAI/bge-base-en-v1.5; agglomerative clustering at a cosine threshold; cross-links to released centroids from the last 14 days so “OpenAI launches X” doesn't reappear three days running. Writes clusters.jsonl + embeddings/centroids.npz. No LLM. ~20–60s.

rank

One LLM call per cluster. Scores 0–100 against config/rubric.yaml: significance · hands_on_utility · big_picture_relevance · financial_services_impact · freshness_momentum. Tags audiences. Assigns tier: pulse / on_the_radar / cut. Writes ranked.jsonl. ~$0.05–0.10 with Sonnet. ~60–90s.

summarise

For top-N stories, fetches the article body via trafilatura, then LLM-drafts: headline, body, direction note, signal pill (act / try / read / watch / discuss). Writes section intros per section. Assembles into the Issue model. Writes issue.json. ~$0.10–0.15. ~60–180s.

render

Jinja2 → HTML. No LLM, no network. Writes docs/staging/<date>.html. aiv release re-renders into docs/released/<date>.html and rebuilds the landing index. <1s.

6. Principles discipline

The four principles that govern engineering decisions in this repo. Any change that violates one of these deserves an explicit Architect sign-off in the PR description.

PrincipleWhat it means in practice
No Token Wasted LLM for judgment only. If code can do it reliably, code does it. The test before any new LLM call: could code do this reliably?
Curated, not aggregated Every issue has a point of view. Arman ratifies. Nothing auto-publishes.
Design-first, eval-first Lock the data contracts before code; build the eval harness before tuning. We optimise against the rubric, not vibes.
Subscribe, don't scrape RSS / Atom / API only. HTML scraping is a rare, isolated fallback. Compliance + fragility both demand this.
