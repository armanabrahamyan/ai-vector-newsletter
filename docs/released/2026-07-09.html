The Pulse · Story of the day

Coding agents can now review and revise their own pull requests

One-shot pull request generation has always been open-loop: propose, then stop. This arXiv preprint introduces SWE-Review, adding a reviewer agent that explores the repository, accepts or rejects the change, and feeds structured feedback into a second pass. The benchmark and training dataset are public. The generate-review-revise loop is now the credible architecture for coding agents handling issue resolution.

The stack has new weak points. Across compute, data, permissions, and shutdown authority, agentic deployments are exposing structural gaps that vendors are racing to fill. The governance question is whether your institution gets there first.

A third of US banks cannot stop an autonomous agent mid-task

Wolters Kluwer's May report finds more than one-third of US banks have no kill switch: a mechanism to pause or disable autonomous AI agents. Regulators are signalling this will be required. If your institution is deploying agentic AI, where does the shutdown authority sit in your governance structure?

Inference costs fell 50x a year, making your data layer the bottleneck

A UC Berkeley perspective piece argues that with inference prices dropping a median 50x per year, the binding constraint on agentic workloads shifts from compute to data architecture. Agents querying databases generate thousands of sub-queries per task, with 80–90% duplicating prior work. Is your data infrastructure designed for a single human analyst, or a swarm?

AI agents routinely grab excess permissions, and safety training won't stop them

An arXiv preprint introduces ToolPrivBench, a benchmark across eight domains, extending 07-06's attack-surface finding: mainstream agents consistently grab higher-privilege tools even when lower-privilege ones suffice. Transient failures amplify the pattern, and general safety alignment doesn't transfer to least-privilege behaviour. If alignment doesn't transfer to least-privilege behaviour, what does your governance framework substitute for it?

Modal's answer is sandboxes, elastic GPU bursting, and feedback loops fast enough for 100,000 isolated RL environments. Hands-On

5 stories

The harness sets the bill. Today's pattern is measurement hygiene: across benchmarks, build tooling, and training data, the shared finding is that what you use to run a model shapes cost and quality as much as the model itself.

Databricks finds token price misleads on coding agent cost

Databricks benchmarked coding agents on real internal pull requests across a multi-million-line codebase. The sharpest finding: Sonnet 5 cost more per task than Opus 4.8 despite cheaper tokens, consuming 1.9x more. Harness choice alone drove 2x cost swings at equal quality. Run task-level cost analysis on your own codebase; per-token pricing will mislead your model selection.

Hugging Face's kernel registry adds code signing and agent-ready build tooling

Hugging Face has redesigned Kernels: custom GPU kernels now live in a dedicated Hub repository type, load only from verified publishers by default, and are signed via Sigstore's ephemeral keys. A new cross-framework interface lets one kernel run across PyTorch, JAX, and CuPy. Pull `kernels` and `kernel-builder`; the signed-publisher default gates arbitrary code before it reaches your process.

Currents

1 item

History is becoming data. Language models are starting to make century-old financial records queryable at scale. The signal is early and the sourcing is thin, but the direction is clear: unstructured archives are becoming structured risk evidence.

The New York Fed mapped US bank runs to 1863 with AI

Early signal that language models can unlock historical financial data at scale: the New York Fed screened 374 million digitised newspaper pages, extracting more than 3,000 bank-run episodes from 1863 to 1934. Database is public at finhist.com/bank-runs. Replicated, it becomes a stress-research template; if the extraction carries undetected bias, any stress model calibrated on the count misreads historical volatility.
