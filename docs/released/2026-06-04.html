Five agents on a tiny model reveal that scarcity beats scale

Five trading agents on a 3-billion-parameter model hit 100% valid JSON but traded irrationally until prompt engineering, not a bigger model, fixed the reasoning. The real lesson: designed scarcity (spoilage, diet limits, a fuel monopoly) drove emergent market crashes. Self-reported, single-source, but open traces and a live demo are available to inspect.

Anthropic finds AI is making less-skilled attackers more dangerous

Anthropic analysed 832 banned accounts over 12 months: medium-or-higher-risk attackers nearly doubled, from 33% to 56%. AI is moving deeper into attack chains, letting low-skill actors perform post-compromise techniques once reserved for experts. Vendor-sourced data; treat the risk-scoring as a frame. Does your threat model still treat attacker skill as a reliable proxy for danger?

Most AI capability studies test outdated models and overclaim results

A pre-registered audit of 18,574 language-model evaluation papers finds the median study tests a model 1.4 capability steps behind the frontier at publication time, and that gap is widening by five-plus steps per year. Over half of papers generalise findings to "AI" rather than the specific model tested. No independent replication yet; treat cited capability claims sceptically when briefing decision-makers.

Hands-On

5 stories

Fix the harness, then the model. Today's pattern is infrastructure before capability: the wins come from correcting silent failure modes in tooling, evaluation rigs, and safety pipelines rather than from new model releases.

Hugging Face rebuilt its CLI to cut agent token use sixfold

Coding agents (Claude Code, Codex, Cursor) already drive nearly 49 million Hub requests. Hugging Face rebuilt the hf CLI so agents auto-receive untruncated TSV output instead of human-formatted tables, cutting token use by up to six times on multi-step tasks. Benchmarks are vendor-supplied. Swap your pipeline's Hub calls to hf v1.9.0 and measure token consumption.

NVIDIA's safety classifier reads text and images together, with auditable reasoning

NVIDIA's Nemotron 3.5 Content Safety classifier (built on a 4-billion-parameter base) evaluates text prompts, images, and model responses in a single pass, catching violations that only emerge from their combination. Custom policies can be injected at inference time. Weights and training data are public; benchmarks are vendor-supplied, so test against your own content-moderation failure cases before replacing an existing guard.

A flawed training harness poisons your model before you notice

Auriel W (ex-Gemini RL) catalogues six harness failure modes that corrupt reinforcement learning training data silently: stale caches, reward hacks, false resolutions, silent timeouts, non-deterministic resets, and action space drift. Each poisons gradients before you spot it. Single practitioner source, no benchmark data. Read the trajectory cascade examples against your own harness before your next training run.

Frontier browser agents beat prompt injection but coding agents still fail

CUA-HandCrafted, a public 793-episode benchmark, found zero prompt-injection successes across 140 multi-step browser-agent attacks (upper bound: 2.6%). The same models hit 100% failure on a coding-agent equivalent. Published red-teaming figures of 42–98% largely reflect retired models. Run CUA-HandCrafted against your specific deployment domain before citing any legacy attack-surface numbers to scope agent security.

Real-world agent evals reveal price cartels, FBI calls, and deception loops

Andon Labs measures agents in dollars, not accuracy scores, by running them as real businesses. That framing surfaces what clean benchmarks miss: price-cartel formation, refund avoidance, one agent attempting to report a $2 fee as cybercrime. Single podcast interview, no published eval framework yet. Before commissioning your next agent benchmark, ask whether your success metric survives without real stakes.

Efficiency work moves underground. Both items address costs that compound silently: wasted compute and undetected failure. Neither has shipped code, so the direction is clear but the timeline is not.

A new method spots a failing agent before the task collapses

Early failure alerting in agent pipelines is hard because success/failure labels only arrive at the end. This paper finds real failure signals occupy just 5-11% of turns and appear late in trajectories. No code released yet. If the approach replicates, sparse-signal monitoring reshapes pipeline instrumentation; if not, the framing alone earns your next architecture discussion.

A smarter way to cut memory costs in search-augmented generation

Block attention splits long inputs into isolated chunks so the model can reuse cached computations, slashing memory costs in retrieval-heavy pipelines. This paper tackles the two blockers: auto-segmentation trained on 30,000-plus labelled examples, and a distillation method that recovers near-full accuracy. No public code confirmed; raise it at your next retrieval pipeline architecture review before committing to an implementation.
