Ramp details how agentic AI handles risk operations in production fintech

Ramp's engineering blog lays out how the fintech is moving risk operations from human queues to autonomous agents in production. NVIDIA's agent governance design moves execution into a managed, auditable workspace

Autonomous agents running for hours across enterprise systems need a different security model than chat. NVIDIA's reference design separates the laptop (display only) from a managed virtual machine where identity, network access, and credentials are enforced. Every action that changes a system requires human approval before execution. Does your agent deployment enforce that boundary, or is execution still running locally?

Nubank rebuilt its chat as shared infrastructure to run AI agents

Nubank separated shared infrastructure (session management, routing, message delivery) from business logic, letting teams deploy independent agents via standardised contracts. An agent selector routes each interaction to deterministic or AI-driven agents. The harder shift was organisational: the platform team stopped owning business rules. If your conversational stack still bundles these, who owns the separation decision?

Cross River Bank's expanded partnership with Stripe covers card issuing for agentic commerce: AI agents authorising purchases on behalf of consumers and businesses. Elastic open-sources a cognitive-science-grounded memory layer for stateful agents

Stuffing prior context into the prompt breaks on cost, latency, and the 'lost in the middle' effect. Atlas fixes this with three memory types from cognitive science: episodic, semantic, and procedural, each in a separate Elasticsearch index; retrieval scores 0.89 Recall@10. Code is on GitHub. Clone it and wire it into your next stateful agent build.
