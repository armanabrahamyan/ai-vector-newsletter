The Big Picture

4 stories

The foundations need checking. Three of today's stories quietly expose the same problem: tools and methods already embedded in production workflows rest on assumptions that narrow empirical testing is now calling into question.

Bank of England finds popular explainability tools mislead on correlated data

The Bank of England's research blog ran 500 simulations and found that SHAP, LIME, and similar feature attribution methods rank inputs correctly only when features are independent. Add mild correlation and rank agreement drops to between 0.3 and 0.8, making explanations actively misleading. If your model-risk sign-off relies on feature attribution, test that assumption before the next supervisory review.

Simulating real deployments predicts model safety before release

Replaying anonymised production conversations with a candidate model before release, a new technique predicted the direction of safety-behaviour changes 92% of the time, against 54% for a standard prompt-based baseline. No public code yet. The harder question for safety leads: does your pre-release review process include realistic traffic, or just curated adversarial prompts?

OpenAI and Broadcom have built Jalapeño, a custom chip designed specifically for running large language models at scale. A careful synthesis of scaling laws sharpens compute allocation decisions

Lilian Weng's synthesis distils three decades of scaling-law research into one reference. The core insight: training loss falls predictably as a power law across model size, data, and compute. NVIDIA's fine-tuning library cuts mixture-of-experts training time by 3.5x

NVIDIA's NeMo AutoModel slots into the standard Hugging Face from_pretrained() call via a single import change, delivering 3.4–3.7x faster training throughput and roughly 30% less GPU memory on mixture-of-experts models. Benchmarks are vendor-supplied. If you're fine-tuning a large mixture-of-experts model this sprint, swap the import and measure against your own throughput baseline before scaling the cluster.
