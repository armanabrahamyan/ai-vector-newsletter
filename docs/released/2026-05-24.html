Feeding PDFs directly to vision models costs more and answers worse

The 'just attach the PDF' shortcut underperforms. A single-author benchmark across 171 questions on image-heavy documents found native vision models scored 52% accuracy at $0.26 per query, while extraction pipelines hit nearly 60% for less. Single Reddit thread, no peer review. Before building document question-answering on vision alone, run this test against your own corpus.

Plumbing before performance. Today's updates are maintenance, not milestones. The pattern is narrow backend fixes that quietly expand which hardware can run which models locally, with no new capabilities attached.

llama.cpp now runs Qwen3.5 with better precision on consumer hardware

llama.cpp build b9297 adds NVFP4 multi-token-prediction scale tensors and links Qwen3.5 multi-token-prediction tensors, improving quantised inference fidelity for those models locally. This is a plumbing update, not a capability leap. If you run Qwen3.5 on local hardware, pull the latest build before benchmarking; otherwise, nothing urgent here.

Local inference on Intel hardware gains support for a newer model architecture

llama.cpp build b9289 extends its Intel SYCL backend to support gated delta networks with sequence lengths greater than one, enabling a broader class of recurrent-style models to run locally on Intel GPUs. Narrow but practical: if you run local inference on Intel silicon, pull this build before testing recurrent architectures.
